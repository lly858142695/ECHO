# ECHO Next 26.5.31 更新日志

时间范围：2026-05-31 ~ 2026-06-01

本次更新聚焦：流式播放稳定性优化、播放质量偏好统一、歌词与恢复体验修复、样式主题增强。

## 功能与体验

- 新增“流式音质偏好”统一记忆能力（`lossless / high / standard / hires`），并抽离为独立偏好模块，支持跨页面一致复用（搜索页 / 歌单页 / 播放详情）。
- 统一默认流式音质为 `lossless`，不再使用 `max` 作为可见值；保留历史值兼容，自动将本地旧 `max` 迁移为 `lossless`，避免升级后行为突变。
- 歌单页与流式搜索页下发播放质量时，默认与当前偏好对齐；“流式音质”设置将持久化到本地后续页面自动复用。
- 歌手详情页转换为使用当前流式音质偏好，避免进入详情页面时音质回退不一致。

## 音频与播放流程

- 改进 HTTP 流暂停/恢复逻辑：暂停时不再直接软暂停原生输出，改为预热解码器与/或输出桥接，恢复时可更快进入播放并减少网络重建开销。
- 增加“暂停后保留解码器预热”路径：可在不改变流畅度的前提下提前准备解码/输出，在恢复时使用已预热流实现更平滑衔接。
- 将无指定音质的播放请求统一补齐为 `lossless`，在主流程、IPC 及播放缓存查找阶段都生效，避免默认质量不一致。

## 远程提供商与回退策略

- 将 Netease、QQ、Kugou 的默认质量回退序列调整为优先尝试更高质量（含 `lossless`）后再降级，提升可播放率与音质上限可达性。
- QQ / Netease / Kugou 测试与回退逻辑同步更新，覆盖未显式传入音质时的默认行为。

## 歌词与状态修复

- 歌词页支持优先使用共享的播放状态快照，尤其在状态刷新未及时返回时也能正确定位当前歌词行。
- 修复播放进度跳转/卡顿场景下的显示延迟，优化高频状态更新下歌词滚动的稳定性。
- 播放按钮恢复逻辑优化：对于可见的暂停态（含流媒体与本地状态不同步场景）可直接恢复，避免误触发重新入队播放。

## UI

- `childrenDoodle` 主题补充更多可视化样式规则（进度条、按钮、标题栏、卡片、图标等），提升该主题的一致性与表现力。

## 备注

- 本版本包含较多测试覆盖更新，主要覆盖音质默认映射、远程播放恢复、歌词状态同步与相关组件行为。
